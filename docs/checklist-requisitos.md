# Checklist del periodo de prueba

## Frontend

- [x] React y Vite.
- [x] Componentes separados por responsabilidad.
- [x] HTML semántico y CSS responsivo.
- [x] Catálogo obtenido desde una API.
- [x] Búsqueda, categorías y carga progresiva.
- [x] Login, perfil y panel administrativo.
- [x] Configuración de rutas SPA para Vercel.

## Backend

- [x] Express con separación de rutas, controladores y middlewares.
- [x] Métodos GET, POST, PUT y DELETE.
- [x] CRUD de categorías y recursos.
- [x] Validación de datos.
- [x] CORS, Helmet, rate limiting y manejador de errores.
- [x] Roles `admin` y `user`.
- [x] Tokens de Supabase Auth verificados mediante JWKS.

## Base de datos

- [x] Supabase PostgreSQL.
- [x] Integración con `auth.users`.
- [x] Tablas públicas `profiles`, `categories` y `resources`.
- [x] Relaciones, restricciones, índices y normalización 3NF.
- [x] Row Level Security y políticas de acceso.
- [x] Script de instalación y script de datos iniciales.

## Entrega

- [x] README con instalación local y despliegue en Vercel.
- [x] Documentación de API.
- [x] Diagrama y descripción de base de datos.
